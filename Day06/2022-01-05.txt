postgres와 mySQL의 차이
postgres는 테이블의 컬럼을 한 번 만들면 위 아래 순서를 변경할 수 없다.
mySQL은 테이블의 컬럼을 한 번 만들면 위 아래 순서를 변경할 수 있다.

primary key
- 테이블 레코드를 찾기 위한 기본키 => 기준이 되는 키
- 행을 기준으로 한다.
기본 키(primary key)는 주 키 또는 프라이머리 키라고 하며, 관계형 데이터베이스에서 조(레코드)의 식별자로 이용하기에 가장 적합한 것을
관계 (테이블)마다 단 한 설계자에 의해 선택, 정의된 후보 키를 말한다.
유일 키는 0~1개 이상의 속성의 집합으로 볼 수 있다. - 위키백과

foreign key
- 참조하는 외부 테이블과의 관계를 가리키는 키
- 행을 기준으로 한다.
외래 키는 참조하는 테이블에서 1개의 키(속성 또는 속성의 집합)에 해당하고, 참조하는 측의 관계 변수는 참조되는 측의 테이블의 키를 가리킨다.
참조하는 테이블의 속성의 행 1개의 값은, 참조되는 측 테이블의 행 값에 대응된다.
이 때문에 참조하는 테이블의 행에는, 참조되는 테이블에 나타나지 않는 값을 포함할 수 없다.
이러한 참조 관계는 2개의 테이블을 연관시키기 위한 관계 정규화의 본질적인 부분이다.
참조하는 테이블의 행 여러 개가, 참조되는 테이블의 동일한 행을 참조할 수 있다. - 위키백과

SQL 쿼리
1. INSERT: 데이터 저장
- INSERT INTO 테이블명(컬럼명) VALUES(등록할 값)
- 테이블명 ex. 번호, 이름, 성별, 1월정산금, 미납금, 핸드폰번호, 단말기금액, 이자액
2. SELECT: 데이터 조회
- SELECT * FROM 테이블명 WHERE 조건절 ORDER BY 정렬
- 가장 복잡하고 정보가 많음, 조건이 붙으면 한 테이블에서 정보를 가져오는 것이 아니라 3개 이상의 테이블에서 정보를 가져옴
- SELECT를 좀 더 적게, 빠르게 하는 것은 DB구조 설계에 달려있다.
3. UPDATE: 데이터 수정
- UPDATE 테이블명 SET 수정할 컬럼명 = 수정할 값 WHERE 조건절
ex. UPDATE user SET name = '홍길동', phone = '010-2222-2222' WHERE idx = 10234;
4. DELETE: 데이터 삭제
- DELETE FROM 테이블명 WHERE 조건절
